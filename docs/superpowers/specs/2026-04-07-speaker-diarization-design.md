# Диаризация спикеров — спецификация

## Цель

Добавить распознавание говорящих (speaker diarization) в whisper-subtitles, чтобы субтитры содержали информацию о том, кто говорит, с возможностью задать имена и цвета каждому спикеру, а также экспортировать цветные субтитры в форматах ASS, WebVTT и SRT.

## Требования

1. **Диаризация всегда включена** — каждая задача определяет спикеров автоматически.
2. **Количество спикеров** определяется автоматически моделью.
3. **Модель диаризации** выбирается пользователем в настройках (по аналогии с моделью Whisper).
4. **Переименование спикеров** — отдельный экран после транскрипции, до скачивания.
5. **Цвета спикеров** — автоматически из палитры, с возможностью ручной смены.
6. **Форматы субтитров:** ASS (основной, с цветами) + SRT (дополнительный, текст с метками спикеров) + WebVTT (просмотр в плеере с цветами).
7. **MKV** — встраивание ASS как основных субтитров, SRT как дополнительной дорожки для совместимости.
8. **Fallback** — если HuggingFace-токен не указан, диаризация пропускается: транскрипция работает как раньше, все сегменты относятся к единственному спикеру, экран `AWAITING_SPEAKERS` пропускается.

## Библиотека диаризации

**pyannote.audio** — зрелая, точная open-source библиотека. Использует PyTorch (уже в контейнере). Модели доступны через HuggingFace (gated, требуют токен и принятие лицензии).

Доступные модели (выбор в настройках):
- `pyannote/speaker-diarization-3.1` — основная, высокая точность
- `pyannote/speaker-diarization-3.0` — предыдущая версия, чуть легче
- `pyannote/speaker-diarization` — базовая версия 2.x, минимальные требования к ресурсам

## Архитектура и поток данных

### Текущий пайплайн

```
Аудио → Whisper (transcribe_worker) → SRT → VTT + MKV
```

### Новый пайплайн

```
Аудио ──┬── Whisper (транскрипция) ──→ сегменты с текстом и таймкодами
        │
        └── pyannote (диаризация) ───→ сегменты со спикерами и таймкодами
                                           │
                    ┌──────────────────────┘
                    ▼
            Объединение (merge) — каждому текстовому сегменту
            назначается спикер по максимальному пересечению таймкодов
                    │
                    ▼
            Статус AWAITING_SPEAKERS — ожидание
            переименования спикеров пользователем
                    │
                    ▼
            Генерация субтитров: ASS + SRT + VTT
                    │
                    ▼
            Встраивание в MKV (ASS + SRT дорожки)
            + создание стриминговой версии MP4 + VTT
```

Whisper и pyannote работают **последовательно** на одном GPU — параллелизм не нужен, пиковое потребление VRAM не суммируется (~3 ГБ Whisper, ~1–1.5 ГБ pyannote).

## Модель данных

### Изменения в `config.json`

```json
{
  "password_hash": "...",
  "model": "large-v3",
  "device": "cuda",
  "language": "auto",
  "diarization_model": "pyannote/speaker-diarization-3.1",
  "hf_token": ""
}
```

- `diarization_model` — выбранная модель диаризации.
- `hf_token` — токен HuggingFace для скачивания gated-моделей.

### Изменения в модели Job (БД)

| Поле | Тип | Описание |
|------|-----|----------|
| `speakers` | JSON (text) | Словарь спикеров: `{"SPEAKER_00": {"name": "Speaker 1", "color": "#FF6B6B"}, ...}` |
| `diarization_segments` | JSON (text) | Сырые сегменты от pyannote: `[{"start": 0.0, "end": 3.5, "speaker": "SPEAKER_00"}, ...]` |
| `merged_segments` | JSON (text) | Объединённые сегменты (текст + спикер): `[{"start": 0.0, "end": 3.5, "text": "Добрый день", "speaker": "SPEAKER_00"}, ...]` — источник для генерации субтитров после подтверждения спикеров |

### Новые статусы

- `AWAITING_SPEAKERS` — задача на паузе, ожидает подтверждения спикеров пользователем.
- `GENERATING` — генерация финальных субтитров (ASS/SRT/VTT) и встраивание в MKV.

Полный жизненный цикл задачи:

```
PENDING → DOWNLOADING → EXTRACTING → TRANSCRIBING → AWAITING_SPEAKERS → GENERATING → COMPLETED
```

Задача ставится на паузу в `AWAITING_SPEAKERS`, пока пользователь не подтвердит имена спикеров. Кнопка «Оставить как есть» подтверждает с именами по умолчанию.

## Бэкенд — новые модули

### `app/tasks/diarizer.py`

- `diarize(audio_path, model_name, hf_token, device)` — запускает pyannote pipeline, возвращает `[{start, end, speaker}, ...]`.
- `merge_transcription_with_diarization(whisper_segments, diarization_segments)` — для каждого сегмента Whisper определяет спикера по максимальному пересечению таймкодов.
- `assign_default_speakers(merged_segments)` — формирует словарь спикеров с именами по умолчанию («Speaker 1», «Speaker 2», ...) и цветами из предопределённой палитры.

### `app/tasks/subtitle_generator.py`

- `generate_ass(segments, speakers)` — ASS с цветными стилями для каждого спикера.
- `generate_srt_with_speakers(segments, speakers)` — SRT с именем спикера в тексте: `[Иван] Текст реплики`.
- `generate_vtt_with_speakers(segments, speakers)` — WebVTT с CSS-классами `<c.speakerN>` для цветов.

Существующая функция `generate_srt_from_result` из `main.py` переносится в этот модуль.

### Изменения в `app/tasks/transcribe_worker.py`

После транскрипции Whisper — запуск диаризации и объединение результатов. Итоговые данные (сегменты с текстом + спикерами, словарь спикеров) возвращаются в `main.py`.

## API-эндпоинты

### Новые

| Эндпоинт | Метод | Описание |
|-----------|-------|----------|
| `GET /api/jobs/{id}/speakers` | GET | Словарь спикеров с примерами реплик (2–3 фрагмента на спикера) |
| `PUT /api/jobs/{id}/speakers` | PUT | Обновить имена и цвета спикеров |
| `POST /api/jobs/{id}/speakers/confirm` | POST | Подтвердить спикеров, запустить генерацию субтитров |
| `GET /api/jobs/{id}/download/ass` | GET | Скачать ASS-файл |

### Существующие (модифицируются)

- `GET /api/jobs/{id}/download/srt` — отдаёт SRT с пометками спикеров в тексте.
- `GET /api/jobs/{id}/subtitles.vtt` — отдаёт WebVTT с CSS-классами спикеров.
- `GET /api/jobs/{id}/download/video` — MKV содержит ASS + SRT дорожки.

## Форматы субтитров

### ASS (основной)

```ass
[Script Info]
Title: Diarized Subtitles
ScriptType: v4.00+

[V4+ Styles]
Format: Name, Fontname, Fontsize, PrimaryColour, ...
Style: Speaker1,Arial,20,&H006B6BFF, ...
Style: Speaker2,Arial,20,&H00FFB347, ...

[Events]
Format: Layer, Start, End, Style, Name, MarginL, MarginR, MarginV, Effect, Text
Dialogue: 0,0:00:01.00,0:00:03.50,Speaker1,Иван,,,,,[Иван] Добрый день, коллеги
```

Каждый спикер — отдельный Style с его цветом. Имя в поле Name и в начале текста.

### SRT (дополнительный)

```srt
1
00:00:01,000 --> 00:00:03,500
[Иван] Добрый день, коллеги

2
00:00:04,000 --> 00:00:06,200
[Мария] Спасибо за слово
```

### WebVTT (для плеера)

```vtt
WEBVTT

STYLE
::cue(.speaker0) { color: #FF6B6B; }
::cue(.speaker1) { color: #4ECDC4; }

00:00:01.000 --> 00:00:03.500
<c.speaker0>[Иван] Добрый день, коллеги</c>

00:00:04.000 --> 00:00:06.200
<c.speaker1>[Мария] Спасибо за слово</c>
```

## Фронтенд

### Экран редактирования спикеров

Отображается при статусе `AWAITING_SPEAKERS`. Для каждого спикера:

- **Цветной кружок** — клик открывает палитру для смены цвета.
- **Текстовое поле** с именем — редактируемое.
- **2–3 примера реплик** — короткие фрагменты из транскрипции для идентификации спикера.

Две кнопки: «Оставить как есть» (подтвердить дефолтные имена) и «Подтвердить» (применить изменения).

### Цветные субтитры в плеере

WebVTT `<track>` с CSS-классами. В `styles.css` — стили `::cue(.speakerN)` с цветами спикеров. Перед текстом реплики — имя спикера в квадратных скобках.

### Локализация

Новые ключи добавляются во все 15 файлов локалей (`app/static/locales/*.yml`): `speakers_title`, `speaker_name`, `confirm_speakers`, `keep_as_is`, `speaker_color`, `hf_token_required` и др.

## Docker и зависимости

### `Dockerfile.base`

Добавить `pip install pyannote.audio`. Зависимости (speechbrain, asteroid-filterbanks и др.) подтянутся автоматически. PyTorch уже есть. Дополнительный размер образа: ~200–300 МБ.

### `requirements.txt`

Добавить `pyannote.audio` (версию зафиксировать при реализации, текущая стабильная — 3.3.x).

### GPU-память

| Этап | VRAM |
|------|------|
| Whisper (large-v3) | ~3 ГБ, выгружается после транскрипции |
| pyannote (diarization-3.1) | ~1–1.5 ГБ, выгружается после диаризации |

Последовательная работа — пиковое потребление не суммируется. GPU с 6+ ГБ VRAM достаточно.

### HuggingFace-токен

Модели pyannote — gated. Пользователю нужно:
1. Зарегистрироваться на huggingface.co.
2. Принять лицензию на странице модели.
3. Создать токен и ввести его в настройках приложения.

Если токен не указан — предупреждение в настройках. Транскрипция работает, но все сегменты относятся к одному спикеру (fallback).
